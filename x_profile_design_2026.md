## 第1章：本設計書の目的と使い方

本書は、2026年のX（旧Twitter）において個人アカウントのプロフィールを設計・最適化し、安定してインプレッションを獲得するための実践的な設計書である。AI（ChatGPT、Claude等）に本書を読み込ませることで、各フィールドのテキストを自動生成できるように構造化されている。

### 使用方法

1. 第3章のテンプレートに自分の情報を埋める
2. AIに「この設計書に基づいてプロフィールを作成して」と指示する

## 第2章：2026年Grok AIアルゴリズムの核心原則

プロフィール設計・投稿設計の前提として、以下のアルゴリズム原則を理解する必要がある。

### ♠ エンゲージメント評価スコア表

| エンゲージメント要素 | スコア | プロフィール・投稿への示唆 |
|---|---|---|
| リプライへの返信（投稿主） | **75.0** | → プロフで「対話重視」を明示し、実行する |
| リプライ（返信） | **27.0** | → リプ戦略と80/20法則をプロフで体現 |
| ブックマーク（保存） | **10.0** | → 「保存推奨」の型で獲得を狙う |
| プロフィール閲覧→フォロー | **12.0** | → 数秒でフォロー決断させる設計が必須 |
| ポスト詳細クリック+2分滞在 | **11.0** | → 固定ポストに「読ませる」コンテンツを配置 |
| 画像・動画の閲覧 | **×2倍** | → ヘッダー・アイコンの視覚的品質が直接影響 |
| 通常リポスト（そのまま拡散） | **20.0** | → いいね20個分。インプレッション爆発の最短ルート |
| 引用ポスト（視点を添えて拡散） | **高重み** | → 「新たな文脈の追加」をAIが高評価。おすすめ優先表示 |
| いいね | **0.5** | → 「いいね」依存の運用は非効率 |

> **重要**: 2026年のアルゴリズムでは「投稿主がもらったリプライに返信する」アクションが、いいね1個の**150倍（75点〜）**という圧倒的な重み付け。単なる「いいね」狙いではなく、**「滞在時間（Dwell Time）」と「リプライの往復」と「リポストの獲得」**を最大化する設計が不可欠。

### ♠ SIMクラスター戦略（Semantic Interest Matrix）

Grok AIはプロフィール文・名前・固定投稿・過去の投稿履歴を統合的に解析し、アカウントの「専門クラスター」を特定する。

**核心ルール:**

- 発信テーマを **1〜3つ** に厳格に絞り込む
- プロフィール全体で一貫性を保つ（名前・自己紹介・固定投稿・投稿履歴すべて）
- 関連キーワードを箇条書きで配置し、AIのキーワード抽出精度を高める

**⚠ 致命的ミス:** 「マーケティング」と「料理」と「政治」を混在させて発信すると、AIはクラスター特定に失敗し、**誰のタイムラインにも表示されなくなる**。

### ♠ AIモデル「Phoenix」の投稿評価プロセス

2026年のXアルゴリズムは、AIモデル「Phoenix」によって投稿から**0.1秒後のユーザー反応を予測**する段階に進化している。

- **1行目** でSIMクラスター（誰に届けるべきか）を瞬時に判断
- **過去128件** のエンゲージメント履歴からユーザー反応を予測してフィードを構築
- **滞在時間** を最重要指標として評価
- **文脈の深さ** を理解し、読後の満足度まで判定
- **For Youフィード** はフォロー中50%＋おすすめ50%の比率で構成
- **4週間ごと** にアルゴリズムを更新（X社公約）

## 第5章：安定して伸びる投稿の型（3パターン）

リサーチ結果に基づき、個人が今日から使える「安定して伸びる投稿の型」を3つ定義する。

### 型1：【結論 → 根拠 → 具体例】リスト形式（保存・再読狙い）

AI（Grok）は投稿の1行目を見て「誰に届けるべきか（SIMクラスター）」を瞬時に判断する。チェックリストやまとめ形式は「ブックマーク（10点）」や「詳細クリック（11点）」を獲得しやすい。

| パート | 内容 | 具体例 |
|---|---|---|
| **1行目（フック）** | 断定的な結論 | 「2026年の副業、正解は〇〇一択です」 |
| **中盤（ボディ）** | なぜそう言えるのか、箇条書きで3〜5つの要点 | 根拠を数字付きで提示 |
| **終盤（アクション）** | 保存・ブックマーク推奨を明記 | 「忘れないように保存推奨」 |

> **ポイント:** テキストだけでなく、**複数枚の画像スライドや図解**を添えると、スワイプによって滞在時間がさらに延び、インプレッションに強力なブーストがかかる。

### 型2：【「続きを読む」誘導】ストーリー形式（詳細クリック狙い）

「表示をもっと見る」をタップさせる行為に高いスコアが割り振られている。140文字以内の短文でも、あえて途中で話を切り、クリックさせる技術が重要。

| パート | 内容 | 具体例 |
|---|---|---|
| **1行目** | 感情を揺さぶる「失敗」や「驚きの結果」 | 「月収300万を失った日の話をします」 |
| **2行目以降** | 核心部分に触れる直前で改行を多用し、「続きを読む」へ視線を誘導 | 行間を意図的にコントロール |
| **ラスト** | 必ず「学び」や「解決策」を提示し、読後の満足度を高める | 具体的な教訓・アクションを明記 |

> **重要:** 「Phoenix」モデルは文脈を深く理解するため、釣りタイトルで終わらせず、**必ず価値のある着地点を用意する**。

### 型3：【二択・質問】対話創出形式（リプライ往復狙い）

「投稿主がもらったリプライに返信する」アクションは **いいね1個の150倍（75点〜）** という圧倒的な重み付け。

| パート | 内容 | 具体例 |
|---|---|---|
| **質問** | 答えやすいオープンエンドの二択 | 「AとB、あなたならどちらを選びますか？」 |
| **運用ルール** | 投稿後2時間はオンラインを維持し、すべてのリプライに質問を混ぜて返信 | 会話を継続させることが最重要 |
| **AI効果** | アルゴリズムがその投稿を「活発なコミュニティ」と認識 | フォロワー外の「おすすめ」へ強力に拡散 |

### 投稿の型：AIプロンプトテンプレート

```
【AIプロンプトテンプレート：投稿作成】

以下の条件でX（旧Twitter）の投稿を作成してください。

■ 投稿の型: ［型1:リスト形式 / 型2:ストーリー形式 / 型3:対話創出形式 から選択］
■ テーマ: ［ここに入力］
■ ターゲット層: ［ここに入力］
■ 伝えたい核心メッセージ: ［ここに入力］
■ 実体験・具体的エピソード: ［ここに入力］

【条件】
・1行目で「誰に届けるべきか」をGrok AIが判断できるよう、専門キーワードを含める
・滞在時間を最大化するため、改行を適切に配置
・型1の場合：最後に「保存推奨」を明記
・型2の場合：核心の直前で「続きを読む」を誘導する構成にする
・型3の場合：答えやすい二択の質問で締める
・外部リンクは本文に含めない（リプ欄に記載する旨を付記）
```

## 第6章：個人の「安定」を支える3つの鉄則

### 鉄則1：「X記事（Articles）」の活用（Premium必須）

2026年1月に全Premiumユーザーへ開放された「記事機能」は、現在**最もインプレッションが伸びやすいフォーマット**。

- 長い滞在時間を確保できるため、**週に1回**は長文の「ストック型コンテンツ」として投稿する
- 専門知識をまとめた記事はSIMクラスターの強化にも直結する
- X Premium加入は2026年の個人運用において事実上必須

### 鉄則2：外部リンクは「初手」で貼らない

本文にURLを貼ると、**拡散力が30〜50%減少する**。

- リンクは必ず **「1つ目のリプライ（自己レス）」** に置く
- またはプロフィールのリンクへ誘導する
- 投稿本文では「ゼロクリックコンテンツ」（リンクなしで価値が完結する形式）を徹底

### 鉄則3：ネイティブ動画（15〜30秒）の添付

外部（YouTube等）のリンクではなく、Xに直接アップロードされた動画が優先される。

| 要素 | ルール |
|---|---|
| 冒頭3秒 | 結論またはインパクトのある映像を提示 |
| 字幕 | 完備必須（無音環境でも価値が伝わる構成に） |
| 長さ | 15〜30秒が最適（短尺動画は完走率60%以上を目標） |
| アップロード方法 | **必ずXにネイティブアップロード**（YouTube等のリンク共有は厳禁） |

> **Build in Public（過程の公開）戦略:** 自分の成功も失敗もリアルタイムで見せるストーリーを上記の型に当てはめると、AIに評価されつつ、熱狂的なファン（コアファン）を作りやすくなる。

## 第7章：リポスト戦略とエンゲージメント基準値

リポスト1件はいいね20個分に相当し、インプレッションを爆発させるにはリポストの獲得が最短ルートとなる。

### 7-1. 「通常リポスト」と「引用ポスト」の使い分け

2026年の仕様では、この2つはAIによって明確に区別して評価されている。

| 種類 | 加点 | 特徴 | AI評価軸 |
|---|---|---|---|
| **通常リポスト** | **20点** | 投稿をそのまま拡散する行為 | コンテンツの「普遍的な価値」が高いと判断。「バイラルループ（拡散の連鎖）」を生むエンジンとして重視。 |
| **引用ポスト** | **高重み付け** | 自分の意見（視点）を添えて拡散する行為 | 「視点の追加」を高く評価。元の投稿に「新たな文脈」を加える引用ポストが多い投稿は、おすすめフィード（For You）へ優先的に配信。 |

### 7-2. エンゲージメント率（ER）の基準値

100万件以上のポストを分析した2026年の統計データに基づく、個人アカウントの成長判断基準。

| フォロワー規模 | 平均ER | 優秀ER |
|---|---|---|
| 1,000人未満 | 0.08% 〜 0.15% | **0.33%以上**（おすすめフィードへの露出が激増） |
| 1,000人〜10,000人 | 0.06% 〜 0.10% | 0.10%以上で成長軌道 |

### 7-3. リポストを誘発する3つの仕掛け

| 仕掛け | 解説 | 効果 |
|---|---|---|
| **「保存・再読」の動機付け** | チェックリスト、比較表、図解など「後で必要になるから自分のタイムラインに置いておきたい」と思わせる形式にする。 | ブックマーク（10点）とリポスト（20点）の同時獲得 |
| **「自己レス（連投）」の活用** | 1枚目の画像で興味を引き、2枚目・3枚目のリプライで深い知識を出すスレッド形式。 | 滞在時間延長＋スレッド全体のリポスト率が**1.5〜2倍**に向上 |
| **セルフリポストの制限** | 自分の投稿をリポストする行為は、2026年の「重複コンテンツ検知システム」によりデブーストリスクあり。 | **1ポストにつき1回、時間を空けて行う**のが安全ライン |

### 7-4. リポストされる投稿の絶対条件

リポストを獲得するには、以下の2つの条件を**両立**させる必要がある。

1. **専門性:** 「これは自分のフォロワーにとっても有益だ」と判断される専門的な価値
2. **自己呈示欲求:** 「この投稿をシェアしている自分は、センスが良い（あるいは情報が早い）と思われたい」とユーザーに感じさせる価値観

> **統計データ:** 2026年、上記の条件を満たす投稿は平均**6.67件のリポスト**を獲得している（2024年から35%増加）。

## 第11章：リスク管理チェックリスト

シャドウバンや可視性制限を回避するための必須チェック項目。

| 禁止事項（ペナルティトリガー） | 対策・推奨行動 |
|---|---|
| 短時間での過剰ないいね・フォロー連打 | 自然なペースで操作。自動化ツールの過度な使用を避ける。 |
| 同一内容の反復投稿 | 毎回ユニークな価値を提供する。コピペ投稿は禁止。 |
| 外部リンクの乱用 | ゼロクリックコンテンツに移行。リンクはリプ欄に置く等の工夫。 |
| ハッシュタグの過多（3〜5個以上） | ハッシュタグは0〜2個に抑える。自然言語でトピックを伝える。 |
| ユーザーからのネガティブフィードバック蓄積 | 「興味なし」「ブロック」を受けない良質なコンテンツ制作を徹底。 |
| セルフリポストの乱用 | 重複コンテンツ検知システムによりデブーストリスク。**1ポストにつき1回、時間を空けて行う**。 |
| インプレッション急落時（シャドウバン疑い） | **48〜72時間の完全冷却**→問題投稿削除→復帰後は良質コンテンツで再開。 |

### ペナルティの種類

| 種類 | 症状 |
|---|---|
| **GhostBan** | 自身のリプライが第三者から完全に見えなくなる |
| **ReplyDeboosting** | リプライがスレッド最下部に沈み、デフォルト非表示 |
| **Visibility Filtering** | おすすめフィードへの掲載停止＋既存フォロワーへの表示抑制 |

## 第12章：プロフィール設計完了チェックリスト

プロフィールを公開する前に、以下の全項目を確認すること。

- [ ] アカウント名に専門キーワードが入っているか
- [ ] 自己紹介文の冒頭で「誰向けか」が明示されているか
- [ ] 実績・数字が1つ以上含まれているか
- [ ] フォローのベネフィットが明確か
- [ ] 人間味・失敗談が含まれているか
- [ ] CTA（行動喚起）が設定されているか
- [ ] アイコンが高画質な顔写真か（400×400px）
- [ ] ヘッダー左側2/3にキャッチコピーがあるか（1500×500px）
- [ ] 固定ポストが「コンピテンシーポスト」か
- [ ] 発信テーマが1〜3つに絞られているか（SIMクラスター）
- [ ] 禁止事項（リスク管理）に投稿が抵触していないか
- [ ] 投稿の型（第5章）のうち、最初に使う型を決めたか
- [ ] X Premiumに加入したか（記事機能の活用のため）
- [ ] リポストされる投稿の2条件（専門性＋自己呈示欲求）を意識しているか（第7章）
- [ ] セルフリポストのルール（1回/時間を空ける）を理解したか（第7章）
