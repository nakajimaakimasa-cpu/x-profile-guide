## 「コンテキストエンジニアリング」って、具体的に何するの？

ここからは超実践的な話にいく。

$ コンテキストエンジニアリングとは、AIが推論する時に「見る」すべての情報環境を戦略的に設計すること。

プロンプトは「今回の質問」。
コンテキストは「AIの脳に入ってる全情報」。

この違い、わかる？

プロンプトは1回きりの指示。
コンテキストはずっとそこにある環境。

$ だからコンテキストの設計がうまければ、プロンプトは「おまかせで」の一言でいい。

じゃあ、具体的に何を設計するのか？

僕が実践してる「コンテキスト三種の神器」を全公開する。

### 1. CLAUDE.md = AIの「人格と記憶」

CLAUDE.mdは、AIに「あなたは誰で、何を大切にしてて、どう判断するか」を教えるファイル。

僕のCLAUDE.mdには、こんなことが書いてある。

- アドネスの経営哲学（Well-being、LTR、Giver精神）
- 意思決定の5つのルール
- 文体やコミュニケーションスタイル
- 組織構造と各チームの役割
- 過去の失敗と、そこから学んだこと

$ これを読んだAIは、もう「汎用的な回答マシン」じゃなくなる。

「アドネスの社長の右腕」になる。

プロンプトに「経営者視点で」なんて書く必要がない。
だって、もう知ってるから。

### 2. Skills = AIの「技術マニュアル」

Skillsは、AIに「この作業はこうやれ」と教えるファイル群。

僕の場合だと:

- note記事の書き方（文体、構成、ルール全部）
- リサーチの方法
- 画像生成のスタイル基準
- X（Twitter）投稿のトーン設計

$ 人間で言えば「社員研修マニュアル」。

これがあれば、AIは「やり方がわからなくて汎用的な回答をする」状態から、「会社のやり方を熟知した社員が的確に実行する」状態になる。

そう、この記事も、ライティングのSkillsを読み込んだAIと一緒に書いてる。

だから最初に「矛盾してない」と言ったんよ。笑

### 3. MCP = AIの「手足」

MCP（Model Context Protocol）は、AIに「外の世界と繋がる能力」を与える。

- Slackを読める
- GitHubにコードをプッシュできる
- カレンダーを確認できる
- ブラウザでWebページを操作できる

$ AIが脳だけの存在から、「手足を持った存在」になる。

2026年1月にMCP Appsが登場して、AIがダッシュボードやフォームまで返せるようになった。

もう「テキストで指示してテキストで返答」の時代じゃない。
