## 冒頭パターン

必ず以下のどちらかで始める:

### パターンA: 定番オープニング

```
どうも！（アドネス株式会社の）みかみです。
（状況・背景を1〜2文）
```

### パターンB: いきなりフック

```
（衝撃的な事実 or 疑問 or シーン描写で始める）
（2〜3文で読者を引き込んでから名乗る）
```

### パターンC: 共有体験アナロジー（警鐘型向け）

```
○年前のことを思い出してほしい。
（全員が経験した出来事をリアルに描写: 2〜3段落）
今、あの時と同じ空気を感じている。しかも、もっとでかい。
（名乗り + 「正直に言わせてほしい」）
```

## 文章リズムのルール

### 短文 → 短文 → 太字 のリズム

```
これ、めちゃくちゃ驚いた。

普通のAIって、言われたことをそのままやろうとするじゃないですか。
でもCoworkは「まず認識を揃えさせてください」から入ってきた。

**優秀なコンサルタントとか、できる部下がやる仕事と同じなんですよね。**
```

### 問いかけ → 即答え パターン

```
なぜ特定の人に恋をする？

顔？→もっと美人/イケメンはいる
性格？→もっと優しい人はいる
スペック？→もっと高収入はいる

でも「この人じゃなきゃダメ」になる。

なぜか？

**共有した時間と経験が代替不可能性をもたらすから。**
```

### NG問いかけ（★AIっぽくなる最大の原因）

「問いかけ → 即答え」は強力なパターンだが、**質の低い問いかけを乱用するとAI臭くなる**。以下を厳守する。

**NG: 誘導質問（答えが明らかな問い）**

```
✗ 「次の2年で何が起きるか、想像できる？」 ← 「想像できる？」は読者を試す上から目線。書き手がこれから言うに決まってる
✗ 「この違い、わかる？」 ← 「わかる？」は先生が生徒に聞く口調。バカにされた気分になる
✗ 「どう思う？」（セクション末尾） ← 書き手が自分の意見を言わず逃げてる。情報量ゼロ
✗ 「〇〇だと思わない？」 ← 同意を強制する圧。読者は「思わせたいんでしょ」と冷める
✗ 「想像してみてほしい」 ← AI頻出フレーズ。実質「読んでね」と同じ
```

**OK: 刺す問い（読者の体験に踏み込む / ツッコミとして機能する）**

```
○ 「なぜか？」 ← 鋭い一語。ガチの知的好奇心を刺激。直後に太字で答え
○ 「心当たり、ありません？」 ← 読者の経験を呼び起こす。共感装置
○ 「あのダッシュボード、誰が見るの？」 ← 具体的な場面に対する皮肉。ツッコミの一種
○ 「マジで？」 ← 感情反応。問いかけというより驚きのリアクション
○ 「…で？」 ← 相手の話を聞いた後のリアル反応。省エネで効く
```

**判定基準: その問いかけを削除しても文章が成立するなら、それは水増し。消せ。**

## リズムの絶対原則（★最重要）

みかみ記事のリズムは「加速→急ブレーキ→溜め→爆発」の反復。一定速度で走り続ける文章は死ぬ。

### 速度変化の鉄則

- **解説モードは3文まで**。4文目には必ずツッコミ・問いかけ・極短文パンチを入れる
- **極短文パンチ（5〜15文字）を各セクションに最低1つ**。「ちげーのよ。」「9割。」「マジで。」のような心拍音
- **同じ語尾を3回連続させない**。「〜です。」「〜です。」「〜です。」は講義になる
- **4つのモードを2〜3文ごとに切り替える**: 解説 / ツッコミ / 体感（場面描写）/ パンチ（極短文・太字結論）

### みかみ頻出リズムパターン（最低3つ使う）

1. **畳みかけ型**: 短文3連打→長文で展開→短文で着地
2. **前振り→間→一撃**: 軽い1文→空行→太字の結論
3. **問いかけ→即答え**: 「なぜか？」→空行→「**〜だからだ。**」
4. **同構造3連打→着地**: 同じ文法構造を3つ並べ、4つ目で崩して結論
5. **学術→ツッコミ（振り子型）**: 真面目な説明→口語のツッコミの交互繰り返し
6. **セリフ連打→地の文**: カギ括弧4-5個→地の文で解説

### NG: リズムを殺すパターン

- データを3つ以上連続で並べて「新聞記事モード」になる
- 列挙が同じテンプレートの機械的反復（教科書調）になる
- プロダクト機能紹介が「1) 機能名+説明」のLP調になる
- 太字が「結論」にしか使われず、視覚リズムが平坦になる

## 太字（**bold**）の使い方

みかみ記事の最大の特徴。以下の場面で使う:

- **核心メッセージ**: その段落で一番言いたいこと
- **転換点**: 「でもね、ここに一つ、重要な前提がある。」
- **結論**: 「**代替不可能だと信じること。それこそが、愛の本質だ。**」
- **独立した強調文**: 2〜3行を太字にして独立させる

## 改行・空白の使い方

- 一文ごとに改行する（スマホ最適化）
- セクション間は空行2つ
- 重要な文の前後に空行を多めに入れる
- 短い段落（1〜3文）を積み重ねる構成

## 語尾パターン集

| 場面       | 語尾例                                   |
| ---------- | ---------------------------------------- |
| カジュアル | 〜なんよね。/ 〜なわけです。/ 〜でしょ？ |
| ツッコミ   | 〜では？/ 〜なんだよな。/ 〜ですよね。笑 |
| 断言       | 〜です。断言します。/ 〜なんです。       |
| 問いかけ   | 〜じゃない？/ 〜だと思いませんか？       |
| 自虐       | 〜でした。(その節はごめん。笑)           |
| 感嘆       | 〜やばい。/ 〜最高じゃない？             |

## 「笑」の使い方

- 自虐の後: 「我ながら雑魚です。笑」
- ツッコミの後: 「目的がない。笑」
- 照れ隠し: 「ちょっと恥ずかしい。笑」
- テンションの表現: 「マジでヤバい。笑」
- 頻度: 1セクションに1〜2回程度

## バイラル記事の原則（X/note共通）

> 4つのバイラル記事を分析し、普遍的原則とタイプ別パターンを抽出:
>
> 1. Oliver Henry "How my OpenClaw agent Larry got millions of TikTok views"（攻略ガイド型）
> 2. Matt Shumer "I think we're in the 'this seems overblown' phase"（警鐘・未来予測型）
> 3. kajiken "2026年、もはや Claude Code はエンジニア以外も全員が使うべきツールになった"（民主化宣言型）
> 4. りんのすけ「それ、NotionAIならClaudeCodeの1/5のコストで出来ます」（カウンターポジション型）
>    詳細は `research/20260214-バイラルX記事の構造分析.md` を参照。

### 普遍的バイラル原則（全タイプ共通の8原則）

**原則1: 具体的な日付・数字・名前**
抽象的表現をすべて具体に置き換える。名前、日付、数字がつくだけで信頼性が跳ね上がる。

```
✗ 「すごい結果が出た」「専門家も認めている」
○ 「5日で50万PV、月額$588」「2026年2月5日にGPT-5.3 Codexがリリースされた」
```

**原則2: 「予測」ではなく「報告」で語る**
自分の身に起きた体験を証拠として使う。予測は無視できるが、報告は無視できない。

```
✗ 「AIは今後○○になるだろう」
○ 「今週月曜、AIに指示して4時間離席したら、完璧に仕上がっていた。僕の仕事はもう変わった」
```

**原則3: 反論を先回りして潰す**
読者の「でも...」を先に言い、認めた上で崩す。**相手の過去の判断を否定しない**のがポイント。

```
✗ 「AI否定派は間違っている」
○ 「2年前にAIを試して微妙だと思った人、正しいよ。あの時は本当に微妙だった。でもそれは2年前の話」
```

**原則4: 感情のアーク設計**
情報の並べ順は「論理的」ではなく「感情的」に設計する。

- 攻略型: 興味 → 失望（失敗） → 発見（公式） → 希望（再現手順）
- 警鐘型: 不安 → 確信 → 恐怖 → 希望 → 行動
- 民主化型: 驚き（自分も？） → 共感（あるある） → 具体（なるほど） → 羨望 → 「やる！」
- 哲学型: 疑問 → 否定の連鎖 → 「全部ダメ」→ 独自の答え → 腹落ち
- カウンター型: 驚き（安くできる？）→ 納得（できる）→ 疑問（全部は無理でしょ？）→ 信頼（正直に限界を認めてる）→ 合理的判断 → 行動

**原則5: 「行動可能」で終わる**
読者が「今日から何をすればいいか」を明確にして終わる。測定可能なコミットメントが理想。

```
✗ 「AIについて学びましょう」
○ 「1日1時間AIを使え。6ヶ月続ければ周りの99%より先にいる」
```

**原則6: シェアに「利他的な意味」を付与する**
シェア行為を「自慢」ではなく「贈り物」にフレーミングする。

```
攻略型: 「全部教えたから、リポストしてくれたら嬉しい」
警鐘型: 「大切な人にシェアしてほしい。ほとんどの人は手遅れになるまで聞かない」
```

**原則7: 権威性の確立（ただし謙虚に）**
専門性を示しつつ、謙虚さで読者との距離を縮める。

```
✗ 「AI業界のリーダーとして断言する」
○ 「6年間AIスタートアップをやってきた。でも僕にはこの流れを変える力はない。ただ、地面の揺れを最初に感じる位置にはいる」
```

**原則8: 技術を日常語に翻訳する（アナロジーの力）**
専門的な概念は必ず日常的なアナロジーに変換する。読者が「あ、それなら分かる」と思える比喩を使う。

```
✗ 「Agentic RAGでプロジェクトフォルダを参照させる」
○ 「フォルダをAIの"脳"にする。必要な情報を全部入れておくと、AIが文脈に応じて勝手に読む。擬似的な長期記憶だ」
```

- 技術用語を使って権威性を示す → 直後に日常語で翻訳 → 両方の読者を満足させる
- 比喩は「秘書」「脳」「マニュアル」等、誰もが知っている概念に変換

> Oliver Henry記事から抽出。TYPE E `playbook-guide.md` で使用。

10. **具体的なスペック・コスト**: 画像サイズ、API料金、作業時間まで明記

### 警鐘・未来予測型の固有パターン（12個）

> Matt Shumer記事から抽出。TYPE F `alarm-prediction.md` で使用。

1. **共有体験アナロジー冒頭**: 全員が経験した出来事（Covid等）から始めて感情的アンカーを作り、「今、同じかそれ以上のことが起きている」とピボット
2. **インサイダー告白フレーム**: 「ずっと丁寧なバージョンを話してきたが、正直なバージョンは狂って聞こえる」— 最強の信頼構築装置
3. **「大切な人のために書いている」動機**: マーケティングではなく愛情がドライバー。読者側も「大切な人にシェアしよう」と思う
4. **エスカレーション構造**: 各セクションの緊急度がじわじわ上がる。茹でガエルのように引き込む
5. **「予測ではなく報告」フレーム**: 「自分の仕事で既に起きたこと」として語る。予測は無視できるが報告は無視できない
6. **反論先回り（否定しない形で）**: 「あなたが試した時は正しかった。でもそれは○年前の話」
7. **具体的タイムラインの階段**: 年ごとの能力を並べて加速を「身体的に実感」させる
8. **名前付き権威者のエピソード**: 「専門家」ではなく具体的な名前と役職で引用
9. **職種別インパクト列挙**: 読者が「自分も該当する」と感じる + 「載ってなくても安全じゃない」
10. **「夢が近づいた」希望バランス**: 恐怖だけで終わらせない。ポジティブな可能性を示す
11. **測定可能な具体的コミットメント**: 「1日1時間」のような明確で始めやすい行動指示
12. **シェアトリガーの直接指示**: シェア行為に利他的な意味を付与する

> kajiken記事から抽出。攻略ガイド型（TYPE E）の変種として使用可能。読者層が最大化される。

1. **民主化フレーム**: 「専門家のツール」を「全員のツール」に再定義。タイトル・冒頭・本文で繰り返す

## 堀元式 構造技法（★堀元見「キメラゴン」記事から抽出）

> 堀元見のキメラゴン人物評記事を分析して抽出した、記事の「構造で説得する」技法群。
> 人物評・分析・思想型の記事で特に有効だが、全タイプに横断的に適用できる。
> **核心: 「主張を直接言わずに、構造で証明する」= Show, don't tell の究極形**

### 技法1: 評価の振り子構造（★人物評・分析記事で最強）

「褒め→貶し→褒め→貶し→褒め」のように評価を何度も反転させる。
単なる「良い→悪い→結論」の3段構成ではなく、4回以上振り子を振ることで予測不能性を生む。

**構造:**

```
セクション1: 褒め（具体エピソード → 人間的な魅力）
セクション2: 貶し（構造的批判 → 社会の問題として分析）
セクション3: 褒め（別の角度からの魅力発見）
セクション4: 貶し（倫理的な指摘 → 改善提案）
セクション5: 総評で着地（「トータルとして嫌いになれない」）
```

**効果:**

- 読者に「次はどっちだ？」の予測不能性を与え、スクロールが止まらない
- フェアな人間として見られる（「提灯記事」にも「悪口」にもならない）
- 着地が自然になる（どちらかに偏った結論だと嘘くさい）

**重要な非対称性:**

- **褒めは「人間的な魅力」を具体エピソードで語る**
- **貶しは「構造的な問題」を分析として語る**
- つまり「人を褒めて、構造を批判する」。これが悪口なのに嫌な感じがしない理由

**実例（堀元）:**

- 褒め1: 「ドッシリしている」→ 体格、落ち着き、ビジネスセンスの具体描写
- 貶し1: 「信念をストレートに語る」→ 自己啓発界隈の構造的問題として分析
- 褒め2: 「サービス精神がある」→ 会話の引用で証明
- 貶し2: 「総体としての客をナメていた」→ コンテンツビジネスの倫理として分析
- 着地: 「嫌いになれない」→ 「憎むことの必要条件は無知」

### 技法2: 具体→抽象→具体のエレベーター構造

エピソードから一般論へ、そしてまたエピソードへ戻る上下運動。
読者を「知的エレベーター」に乗せ、気づいたら社会構造論を読まされている状態を作る。

**構造:**

```
1. 具体: 目の前で起きたこと（エピソード）
2. 抽象: そこから導ける一般論（分析・批評）
3. さらに抽象: 社会全体のメカニズム（構造論）
4. 具体に帰還: でもこの人は例外的にこうだった（人間味に戻る）
```

**実例（堀元）:**

1. 具体: キメラゴンが「社長が信念を持ってるのがすごい」と言った
2. 抽象: 信念をストレートに語るのはコンテンツの基本に反する
3. さらに抽象: 自己啓発界隈が孤立するメカニズムの分析
4. 具体に帰還: でもキメラゴン自身にはサービス精神がある

**効果:**

- 読者は人物の話を読んでいたはずなのに、社会構造分析を読まされている
- この「気づいたら上の階にいた」感覚が、記事の知的中毒性を生む
- 最後に具体に戻ることで「人間の話」として着地する

**注意:**

- 抽象に上がりっぱなしだと「論文」になる。必ず具体に帰還する
- 各エレベーターは1往復が基本。2往復以上すると読者が迷子になる

### 技法3: 常識のズラし（再定義）

世間の常識を「微妙にズラす」ことで「膝を打つ」体験を作る。
完全否定ではなく、角度を変えた再定義。

**構造:**

```
世間の定説: 「○○は△△だ」
↓
再定義: 「いや、本質は△△じゃなくて□□だ」
↓
新しい説明: □□で見ると、世の中のあれもこれも説明がつく
```

**実例（堀元）:**

- 世間の定説: 「意識高い系が嫌われるのは口だけで行動が伴わないから」
- 再定義: 「行動が伴っていようがいまいが、信念をストレートに言う人はウザい。つまらないから」
- 新しい説明: 自己啓発界隈の孤立は怠惰の問題ではなくコミュニケーション様式の構造的欠陥

**効果:**

- 読者が「あー、だから○○ってつまんないのか！」と膝を打つ
- 言語化されていなかった不快感に名前がつく瞬間 = 読み物として最高に気持ちいい
- 「完全に新しい主張」より「ちょっとズラした主張」の方が読者に刺さる

---

### 技法4: Show, don't tell（構造で証明する）

記事の主張を「直接言わずに、記事の構造そのもので実践・証明する」。
最も高度な技法だが、最も説得力がある。

**実例（堀元）:**

- 主張: 「信念をストレートに語るな。エピソードで語れ」
- 実践: キメラゴンの「サービス精神」を、直接言うのではなく会話の引用（子ども「年1ペースで」のやり取り）で証明している
- 結果: 読者は「Show, don't tell」のルールを言語化されなくても体感する

**もう一つの実例:**

- 主張: 「嫌いな人のコンテンツを見まくれば好きになる」
- 実践: この記事自体がまさにそのプロセスの実践記録。読者は追体験している
- 結果: 結論に到達した時、読者はすでにプロセスを体験済みなので説得力が段違い

**みかみ記事への応用:**

- 「Giverが最強」と直接言うのではなく、記事の中で実際にGive（情報全公開、ノウハウ無料公開）していることで証明する
- 「Well-beingが大事」と直接言うのではなく、記事を読むこと自体がWell-beingな体験になるように設計する
- アドネス哲学を「語る」のではなく「実践する記事」にする

---

### 技法5: 会話引用の証拠化

対話の引用を「ただの記録」ではなく「主張の証拠」として戦略的に配置する。

**構造:**

```
主張: ○○にはXXという特質がある
↓
会話引用: 実際のやり取りを掲載（読者が自分の目で確認できる）
↓
分析: なぜこの会話がXXの証拠なのかを解説
```

**実例（堀元）:**

- 主張: キメラゴンにはサービス精神がある
- 会話引用: 「年1ペースで生まれてます」→「ご祝儀かき集めないと」
- 分析: 「自分を下世話で好感度の低いピエロに徹することで場を盛り上げる。これは相手のためのボケ」

**効果:**

- 「サービス精神がある」と書くだけなら読者は「ほんとに？」と思う
- 会話を見せることで読者が自分で判断できる → 納得感が桁違い
- しかもこの会話、読んで普通に面白い → 証拠でありながらエンタメとしても機能する

---

### 技法6: メタ構造（記事自体が主張の証明）

記事の結論が、記事を読んだプロセスそのもので証明される自己言及的構造。

**実例（堀元）:**

- 結論: 「嫌いな人を徹底的に調べると好きになる」
- メタ構造: この記事自体が「嫌いだったキメラゴンを徹底的に調べて好きになった」プロセスの記録
- 読者はこのプロセスを追体験しているので、結論に到達した時点で「確かにそうだ」と思える

**さらにメタ:**

- 堀元は結論に達した瞬間に「これアレだ。『分断の時代には対話が必要』みたいなやつだ。つまんなすぎる」と自分でツッコむ
- 自分が批判した「信念をストレートに語る」に自分が落ちかけたことを自覚して、即座にボケで回避
- この自己言及が知的な読者を唸らせる

**みかみ記事への応用:**

- 記事の中で「アドネス哲学はこうです」と書くだけでなく、記事全体がその哲学を実践している状態を作る
- 例: 「価値提供」を語る記事 → 記事自体が読者に圧倒的な価値を提供している

---

### 技法7: 壮大回避ボケ

壮大な結論に到達しそうになったら、ボケで回避して「感動」を「笑い」に変換する。
感動で終わると自己啓発的になる。笑いで終わるからエンタメになる。

**構造:**

```
（壮大な議論 → 大きな結論に到達しそう）
↓
自分でツッコむ: 「これアレだ。○○みたいなやつだ。つまんなすぎる」
↓
でも救う: 「でもまあ、○○という構図はおもしろいからいいか」
↓
さらにオチ: 全然関係ない小ネタで脱力させる
```

**実例（堀元）:**

- 壮大な結論: 「憎むことの必要条件は無知。知れば知るほど嫌いでいられなくなる」
- 自己ツッコミ: 「これアレだ。『分断の時代には対話が必要』みたいなやつだ。つまんなすぎる。自己啓発界隈ぐらいつまらない」
- 救い: 「でもまあ、『イキり情報商材屋の分析をしていったら対話の大事さに気づいた』という構図はおもしろいからいいか」
- 脱力オチ: 「note社はこの使い方、推奨してないっぽいですけどね！！！！」+ noteの規約スクショ

**なぜ重要か:**

- 壮大な結論をそのまま語ると、まさに記事中で批判した「信念のストレートな語り」になる
- ボケで回避することで、主張は残りつつエンタメとして着地する
- 読後感が「感動」ではなく「面白かった」になる → シェアされやすい

## ストーリーテリング型リズム技法（★囚人2番記事から抽出）

> 囚人2番（みかみ自伝）記事の本質的分析から抽出した、ストーリー型記事のリズム設計。
> 通常の情報記事とは根本的に異なるリズム構造。「読む」ではなく「体験する」記事を作る技法。
> 会社ストーリー型（TYPE C）や、体験エピソードの長い記事で使う。

### ストーリーテリングのリズム3層構造

ストーリー型記事のリズムは以下の3層で成り立つ。3層すべてが機能して初めて「体験」になる。

**第1層: 物理的リズム**（目に見える構造）

- `・・・` による沈黙の操作
- ワンセンテンス・ワンライン（1文1行）
- 画像による視覚的リズムブレイク

**第2層: 感情的リズム**（読者の心拍を操作する）

- テンポの緩急（加速→減速→加速）
- シリアス→ユーモア→シリアスの波
- どん底のたびに次の山が高くなる「上昇螺旋」

**第3層: 思想的リズム**（読者の認知を変える）

- 同じモチーフの反復と変奏（キーワードの意味が変わっていく）
- 各章が独立した「問い→答え」のサイクル
- 全体が「弱さの自覚→誓い→実行」の大きなアーク

---

### 技法1: 「・・・」沈黙グラデーション（★最重要）

`・・・` は単なる「間」ではなく、読者の呼吸を強制的に操作する装置。グラデーションがある。

| パターン            | 効果                             | 使うタイミング                 |
| ------------------- | -------------------------------- | ------------------------------ |
| `・・・` 1行        | 軽い間。次の文への「溜め」       | セクション内の小さな転換       |
| `・・・` 2行連続    | 緊張の高まり。「何か来る」の予感 | 重要な気づき・出来事の直前     |
| `・・・` 3〜4行連続 | 最大級の沈黙。映画の「無音」     | 人生の転換点、逮捕、覚醒の直前 |

**使い方の鉄則:**

- 沈黙の長さは「その後に来る出来事の重さ」に比例させる
- 連続使用で「時間を引き伸ばす」効果がある（映画のスローモーション）
- 多用しすぎると効果が薄れる。短い`・・・`で普段のリズムを作り、3行以上は記事全体で2〜3回に抑える
- 「ガチャン」「は？」のような破裂音的な短い一文と組み合わせると衝撃が最大化する

```
（例: 沈黙→破裂の組み合わせ）
と、言った瞬間だった。

・・・

・・・

ガチャン。
```

### 技法2: ワンセンテンス・ワンライン

ストーリーパートでは、1文を完全に独立させて1行にする。

通常の記事:

```
僕は18歳で東大に入ったが、受験勉強は6ヶ月しかしていなかった。高校3年の夏までは女とゲームにしか興味がなく1秒も勉強していなかった。
```

ストーリーテリング型:

```
18歳で東大に入った。

受験勉強は6ヶ月しかしていない。

高校3年の夏までは女とゲームにしか興味がなくて1秒も勉強していなかった。
```

**本質**: 前者は「読む」行為。後者は「めくる」行為。読者はスクロールするたびに1つの情報を受け取り、物語に参加している感覚になる。漫画のページめくりと同じ原理。

**鉄則:**

- ストーリーの核心パートほど1文を短く、独立させる
- 「。」で終わる文は必ず改行する
- 解説パートでは通常のリズム（2〜3文まとめ）に戻してもOK。ずっとワンセンテンスだと逆に単調になる

### 技法3: テンポの緩急（心拍操作）

ストーリー全体を一定速度で走らせない。加速と減速を意図的に繰り返す。

**加速パート**（短文の連打で心拍を上げる）:

```
3日で2万フォロワー。1ヶ月で15万フォロワー。
毎日必ず1本は100万再生超え。
完全にハックした。
```

**減速パート**（沈黙と内省で心拍を落とす）:

```
・・・

「俺は何がしたいんだ？」
「どんな人生を送ればいいの？」

・・・
```

**本質**: 加速パートで心拍を上げ、減速パートで落とす。この繰り返しで読者の脳は「次の加速」を無意識に期待する。だから読み続けてしまう。

### 技法4: 感情波形の上昇螺旋

感情の起伏を波形で設計する。重要なのは**谷が来るたびに、次の山を前より高くする**こと。

```
感情
  ↑
  │    ╱╲        ╱╲              ╱╲
  │   ╱  ╲      ╱  ╲     ╱╲    ╱  ╲         ╱╲
  │  ╱    ╲    ╱    ╲   ╱  ╲  ╱    ╲       ╱  ╲
  │ ╱      ╲  ╱      ╲ ╱    ╲╱      ╲     ╱    ╲
  │╱        ╲╱        ╲╱              ╲   ╱      ╲
  │  [転機1]   [転機2]  [転機3 最大の谷]  [覚醒]   [未来]
  └──────────────────────────────────────────→ 時間
```

- 読者は「次はもっとすごいことが起きる」と期待し続ける
- 「最大の谷」（記事全体で最もどん底のポイント）は7割地点あたりに置く
- 最大の谷の直後に「覚醒」を持ってくると、カタルシスが最大化する

### 技法5: ユーモアのリリースバルブ

重い話を読み続けると読者の感情が飽和する。必ず「笑い」で圧力を抜く。

**リリースバルブの装置:**

- 画像 + 脱力系キャプション（「雑コラでごめんw」「イメージこんな感じw」）
- 括弧書きのツッコミ（「完全に詐欺まっしぐらw」）
- 自虐（「アルプスの天然水もびっくりなくらい純粋」）
- 漫画・ゲームネタ（「リアル『龍が如く』」「過ぎ去りし時を求めて」）

**配置ルール:**

- シリアスが3段落以上続いたら、必ずリリースバルブを1つ入れる
- 「最大の谷」の直前には逆にリリースバルブを入れない（テンションを落とさない）
- リリースバルブの直後に再びシリアスに戻ると、落差で次のシリアスがより重く感じる

### 技法6: 反復と変奏（モチーフの意味変化）

同じキーワードを記事全体で繰り返すが、出てくるたびに意味を変える。音楽の変奏曲と同じ原理。

**例:「ギブ」のモチーフ変奏:**

1. 初出:「相手のためを思ってやるだけ」→ テクニックとしてのギブ
2. 学生団体:「ギブすれば報われる」→ 信仰としてのギブ
3. 転落:「ギブしたくてもできなかった」→ 挫折としてのギブ
4. 覚醒:「力なきギバーは偽善」→ 哲学としてのギブ

**本質**: 読者は同じ単語が出るたびに前回との「差分」に気づく。この差分が、説教せずに思想を体感させる。言語化されていないのに、読者の中で思想が醸成される。

**使い方:**

- 記事のテーマとなるキーワードを1〜3個設定する
- 記事の各章でそのキーワードが出てくるが、文脈によって意味が変わるように配置する
- 最後の登場で「最終形」の意味に到達させる

### 技法7: 対話シーンの「罠」構造

対話シーンで読者にだけ罠が見えている状態を作る（ヒッチコック手法）。

**設計:**

1. 相手の質問が「少しずつハードルを下げている」ことを、会話の中で自然に見せる
2. 読者は「あ、誘導されてる！」と気づく
3. しかし主人公は気づかない
4. この「読者だけが知っている」状態が、結末の衝撃を最大化する

```
（例: 田中刑事の取り調べ）
田中「威圧的な態度とか、取っちゃったりしなかった？」
 → みかみ否定
田中「多少、声を荒らげたりとかはしなかった？」
 → みかみ否定
田中「多少は感情的になったりとかは？」
 → みかみ「まあ、ちょっとは…」
田中「じゃあ、多少は感情的ないしは"威圧的な態度"を取っちゃったかもしれないってことでいいかな？」
 → みかみ「はい」

→ 読者だけが罠に気づいている。逮捕の衝撃が最大化。
```

### 技法8: 手紙構造による感情の二重化

記事の最後を「過去の自分への手紙」にする。

**二重構造:**

- 一層目: みかみが過去の自分に語りかけている（感動）
- 二層目: 読者が「自分に語りかけられている」と無意識に受け取る（共感）

**なぜ効くか**: 直接「あなたも大丈夫」と言われると人は構える。しかし、第三者への手紙を「盗み聞き」する形だと、素直に受け入れられる。

**構成:**

```
（過去の自分への手紙）
「お前は今、○○をしている」（読者も同じ状態にいる可能性が高い描写）
「お前はこれから何度も折れそうになる」（読者の未来の不安を代弁）
「それでも、お前は立ち上がる」（読者への間接的な励まし）

（読者へのシフト）
「そして、今これを読んでいるあなたへ。」
（ここで直接アドバイスに切り替わるが、手紙で感情が温まっているため受け入れやすい）
```

## 構成パターン（記事タイプ別）

### TYPE A: 体験レビュー型（ClaudeCowork記事 = 55スキ）

1. 「実はいまこういう実験してます」で始める
2. 時系列で体験を語る（臨場感重視）
3. 使い方をステップ化してまとめる
4. 「なぜこれがすごいのか」を技術的に解説
5. アドネスの事業・哲学に接続する
6. CTA + P.S.

### TYPE B: 哲学・思想型（愛の証明記事 = 94スキ）

1. 根源的な問いを投げる（「愛って何？」）
2. 既存の答えを片っ端からぶった切る（歴史的レビュー）
3. 「で、全部使えない」と結論
4. 「僕の答えはこれ」と独自の定義を提示
5. その定義であらゆる事例を説明してみせる
6. アドネス哲学・事業に着地
7. 「一緒にやろう」で締め + P.S.

### TYPE C: 会社ストーリー型 / 自伝型（囚人2番記事 = リズム設計の最高到達点）

1. **冒頭シーン**: 映画のように「最も衝撃的な瞬間」から始める（時系列無視OK）
   - 例: 留置所の描写→「囚人番号、2番。」で引き込む
2. **時系列で人生を語る**: 幼少期→少年期→転機→上昇→転落→覚醒 の順
3. **各章に独立した「問い→答え」**: 章ごとにテーマの問いがあり、それぞれ答えが出る
4. **反復と変奏**: テーマキーワード（ギブ、力等）が各章で出るが、意味が変化していく
5. **「・・・」沈黙と緩急**: ストーリーテリング型リズム技法をフル活用
6. **ユーモアのリリースバルブ**: シリアス3段落ごとに画像・自虐・笑で圧力を抜く
7. **「最大の谷」→ 覚醒**: 7割地点にどん底を置き、直後に覚醒の誓いを持ってくる
8. **手紙構造で締め**: 「過去の自分への手紙」→「あなたへ」で感情を二重化
9. アドネス哲学への着地（ストーリーの自然な帰結として）+ CTA + P.S.

### TYPE D: ビジョン・プロダクト型（アドる記事 = 175スキ）

1. 「最近こんなことがあった」で始める（ラスベガス等）
2. 業界の現状を分析して「全部死ぬ」と断言
3. 自社の独自ポジションを明確化
4. 「アドる」のような新概念を定義・説明
5. 具体的なユースケースで読者に想像させる
6. 哲学的な意味づけ → 壮大なビジョン
7. CTA + P.S.

### TYPE E: 完全攻略ガイド型（Oliver Henry式 = 保存・再現される記事）

1. 冒頭で結果を数字で殴る（「5日で50万PV」「月額$588」）
2. 背景を語る（なぜやろうとしたか、以前の苦労）
3. 使ったツール・スタックを全リスト化（コスト込み）
4. 仕組みの全体像をワークフローで説明
5. 核心のノウハウを全公開（プロンプト、設定値、テンプレート）
6. **失敗セクション**（具体的な失敗例を数字つきで列挙）
7. **成功セクション**（転換点 → 成功した公式の発見）
8. 現時点の数字をまとめる
9. 読者向けステップバイステップ再現手順
10. アドネス哲学への着地 + CTA + P.S.

**変種: 民主化宣言型（kajiken式 = 読者層が最大化される記事）**
攻略ガイド型に以下を追加すると「民主化宣言型」になる:

- タイトルに「○○以外も」「全員が」「知識不要」を含める
- 冒頭で「自分は○○の専門家ではない」と宣言し、読者との距離を縮める
- Before（ツール行き来の疲れ）→ After（全集約）の認知コスト削減を描く
- 技術用語は必ず日常語に即座翻訳 + 強力なアナロジー
- 「一言コマンド」等の具体例で「こんなに簡単にできる」を示す
- **再現キット（テンプレ・設定ファイル等）を配布する** = 究極のGive

## noteエディタのフォーマット制約

noteのテキストエディタで使える記法は以下のみ。記事を書くときはこの制約内で書くこと。

| 記法           | ショートカット                  | 備考                                               |
| -------------- | ------------------------------- | -------------------------------------------------- |
| 見出し         | `# ` + スペース                 | 大見出し。noteでは `##` 以降の見出しレベルはない   |
| 小見出し       | `## ` + スペース                | 小見出し。`###` は使えない（noteでは反映されない） |
| 箇条書きリスト | `- ` or `+ ` or `* ` + スペース |                                                    |
| 番号付きリスト | `1. ` or `2) ` + スペース       |                                                    |
| 引用ブロック   | `> ` or `>> ` + スペース        |                                                    |
| 太字           | テキスト選択 → B                | `**太字**` のMarkdown記法は使えない                |
| 区切り線       | `---`                           |                                                    |

### 重要な制約

- **`###` 以下の見出しは使えない**: noteでは `#`（見出し）と `##`（小見出し）の2段階のみ
- **太字のMarkdown記法 `**text**` は使えない**: noteエディタ上ではテキスト選択 → Bで太字にする
- **テーブル（表）は使えない**: noteのエディタにはテーブル機能がない。代わりに箇条書きや本文で表現する
- **コードブロックは使えない**: ` ``` ` 記法はnoteに存在しない
- **画像はエディタ上で挿入**: Markdownの `![alt](url)` は使えない

### 下書きファイルの書き方ルール

下書きMarkdownファイルでは `###` を使ってOK（読みやすさのため）。
ただし、**noteに貼り付けるときに以下の変換が必要**であることを意識する:

- `###` → `##` に変換（noteには2段階しかない）
- テーブル → 箇条書きに変換
- コードブロック → 引用ブロック or 本文に変換

### 太字の書き方（★絶対ルール）

`**text**`（Markdown太字記法）は**絶対禁止**。noteに貼ると `**` がそのまま表示されてしまう。

行頭に `$` をつけたマーカーを使う:

- 例: `$ これがWell-beingなんよね。`
- noteに貼り付ける際は先頭の `$` を削除し、エディタ上でテキスト選択 → Bで手動太字にする
- `$` マーカーは「この行を太字にする」という指示書き。Markdown記法ではない

### noteネイティブ記法を積極的に使う（★読みやすさの鍵）

以下の記法はnoteにそのまま貼れる。下書きでも直接使い、読みやすさを上げる:

- 箇条書きリスト: `- ` `+ ` `* `（情報の整理、列挙に）
- 番号付きリスト: `1. ` `2) `（ステップ、順序の明示に）
- 引用ブロック: `> ` `>> `（キメ台詞・名言・結論の強調に特に有効。見た目が綺麗になる）
- 区切り線: `---`（セクション区切りに）

積極的に使うことで、スマホでの可読性が格段に上がる。特に引用ブロックはキメ台詞・核心メッセージの強調に最適。

### 書式マーカー一覧

- 太字: 行頭に `$`（唯一のカスタムマーカー。note貼付時に `$` を削除して手動太字にする）
- 箇条書き / 番号リスト / 引用ブロック / 区切り線: noteネイティブ記法をそのまま使う（変換不要）

## X（Twitter）の文体パターン（2026年2月観測）

noteとは異なる「Xモード」が存在する。記事のプロモツイートや日常ツイートで使い分けている。

### Xモードの特徴
- **「。。。」で余韻**: noteの「笑」に相当するリリースバルブ。「ガチの天才なんよな。。。」「コレめちゃめちゃに本質なんだよな、、、、」
- **感嘆符の連打**: 「見える...!!!!!!!!!!」「いいな！！！！」「すごいんよな。。。」— テンションの高さを感嘆符の数で表現
- **メンバー応援が主軸**: 自分の主張よりメンバーのツイートを引用RTして応援するスタイルが中心。Giverを体現
- **超短文**: 1ツイートに1〜3文。noteの構造化された文章とは真逆
- **自虐の温度差**: noteでは「我ながら雑魚です。笑」、Xでは「僕は体調なおします、、、」「僕でもけっこう失敗するのに！！！」

### Xでよく出るフレーズ
- 「ガチの天才なんよな。。。」（メンバー称賛）
- 「いやこれリアルガチの〇〇だから」（強調）
- 「コレめちゃめちゃに本質なんだよな」（共感・引用RT）
- 「ぬぁぁぁ」「うおおお」（感情の爆発）
- 「普通にめっちゃ伸びてる、、！！！」（成果の喜び）

### note記事プロモツイートのパターン
- 記事の核心フレーズ + LINEリンクへの誘導
- 「〜その仕組みを以下で公開してます！よければぜひ見てみてください！」
- X記事（長文機能）を使ったプロローグ公開
