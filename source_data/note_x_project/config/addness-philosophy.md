## 思想の中心

> **「ググるとわかる。アドればできる。」**

Google検索は「知る」を民主化した。Addnessは「できる」を民主化する。
人類は「知っている」と「できる」の間にある深い谷にずっと苦しんできた。
情報はもう足りている。足りていないのは、「知」を「行動」に変換する力。
それが「アドる」であり、Addnessの全てはこの一行に集約される。

## コア概念

### 1. Well-being（5つの軸）

幸福は「欲求の解消」だけではない。

| 軸 | 意味 |
|---|------|
| 欲求の解消 | 欲求を満たすこと |
| 意義性 | 自分の今この瞬間が「目的」にプラスに寄与すること |
| 没頭 | 自分の今この瞬間に集中すること |
| 成長実感 | 過去を振り返り「目的」に対して前進していると感じられること |
| 関係性 | 欲求の源である関係性がきちんと存在していること |

### 2. LTR（Lifetime Relation）

- 継続する関係こそが最大の資産
- 一度つながった関係を命に代えても守る
- 相手にかけてもらったコストを命に代えても返す
- 長く続く関係は新たな欲求を生み、代替不可能な関係へ育ち、**愛**になる
- LTRを紡ぐ手段 = Win-Winの価値提供を維持し続けること

### 3. Giver / Taker

- **Giver**: 価値を与えようとする人。相手をwinさせることを第一優先とする
- **Taker**: 価値を得ようとする人。自分をwinさせることを第一優先とする（相手のloseを厭わない）
- Giverが構造的に報われる世界を作ることがアドネスの存在意義

### 4. 愛 = 代替不可能性

- 愛とは希少性が極大化した時に感じる対象の価値の爆発的な増加
- 「コレは唯一だ！」と信じられた時、そのモノがかけがえのない大切なものになる
- 代替不可能性をもたらすのはLTR（時間の共有）
- 愛の敵は代替可能性（スワイプ文化、転職カルチャー、SNS比較）

### 5. 価値提供

- ビジネスは最も純粋な価値提供の形
- 価値の非対称性を利用してWin-Winを作る
- 自分のGiveが相手に与える価値 > 相手がかけるコスト を常に成り立たせる
- 価値提供 → LTR → Well-being → さらなる価値提供 の無限ループ

### 6. アドる（Addness it!） ── 思想の中心概念

- **「ググるとわかる。アドればできる。」** ── これがアドネス哲学の一行要約
- 「ググる」が情報革命だったように、「アドる」は実行革命
- 「知ってる」と「できる」の間にある深い谷を埋める唯一の動詞
- 目標を持った瞬間に、実行に移すまでの正しい思考プロセスをAIと共に最高効率で実現する行動
- 生成AIの思考リソースを「実行」に変換する装置

## 5つのルール

1. **今、Well-beingする** — 未来のために今を犠牲にしない
2. **命に代えてもLTRを紡ぐ** — 一度つながった関係は命に代えても守る
3. **完璧にこだわる** — 永遠に到達できない「完璧」を追い求め続ける
4. **視点取得する** — 相手の目で見て、相手の心で感じる
5. **Give Super Hard Forever** — 価値提供ゲームのガチ勢で在り続ける

### Addness（AI行動管理OS）
- 「話すだけでやるべきことが明確になり、気づいたら前に進んでいる」を実現
- 個別の機能名は定義していない。以下の5つのケイパビリティで構成される:
  1. ゴールをツリー構造で構造的に分解する（話すだけでゴール→サブゴール→タスク→サブタスクのツリーを生成）
  2. ゴールに繋がっていないタスクを存在させない（ツリー外のタスクは排除される設計）
  3. 今やるべきタスクをAIが自動で選出する（ツリー構造から優先度を判断し、意思決定コストをゼロにする）
  4. ツリーの正しさをAIが常に検証・更新する（前提変化を検知し、静的な計画ではなく動的な構造体として維持）
  5. 個人のツリーを組織のツリーのサブツリーとして接続する（社長→部門長→メンバーの再帰的階層構造）
- 法人・チーム利用から個人の仕事・学習・創作・人生設計まで対応
