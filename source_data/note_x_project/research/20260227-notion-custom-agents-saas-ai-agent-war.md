# Notionカスタムエージェント × 全SaaS AIエージェント戦争 リサーチメモ

## 概要

2026年2月24日、NotionがカスタムエージェントをNotion 3.3として発表。24/7自律稼働するAIエージェントを、ノーコードで構築可能。しかしこれはNotionだけの話ではない。Salesforce、Microsoft、SAP、Adobe、ServiceNow… 全SaaSが一斉にAIエージェントを搭載し始めた。「AIエージェント搭載」はもはや差別化要因ではなく、SaaS業界全体の「断末魔」の可能性がある。

## Notionカスタムエージェントの基本情報

- 発表日: 2026年2月24日（Notion 3.3）
- ステータス: パブリックベータ（5月3日まで無料、以降従量課金）
- 対象プラン: Business / Enterprise
- 料金: Notionクレジット制（$10/1,000クレジット）
- 早期テスターが21,000以上のエージェントを作成
- 主な連携: Slack、Figma、Mail、Calendar、Linear、GitHub、HubSpot、MCPサーバー

## Notionカスタムエージェントの主な機能

1. Q&Aエージェント: 繰り返し質問に自動回答（Notion+Slack+Mail等の情報を活用）
2. タスクルーティング: リクエストの自動振り分け（>95%の精度、25%以上の自律解決）
3. ステータスレポート: 日次/週次/月次レポートの自動生成
4. カスタム構築: 自然言語で記述、テンプレートから構築可能
5. セキュリティ: 全実行ログ記録、権限体系準拠、変更取り消し可能
6. AIモデル選択: Claude Opus 4.6を含む複数モデル対応

## 早期導入企業の成果

- Remote: ITチケットトリアージ20時間/週削減、95%以上の精度
- Ramp: 製品Q&Aを毎日数十件自律回答
- マーケティングチーム: 日次更新レポートで20分/日以上節約
- Braintrust: 6-20時間/週の時間削減

## 全SaaS AIエージェント戦争の全体像

- Salesforce: Agentforce 360（プラットフォーム全体をエージェント中心にリブランド）
- Microsoft: Agent 365（GPT-5統合）
- ServiceNow: AI Agents（Gartner 2025年1位）
- SAP: Joule
- Adobe: AI Foundry
- Workday: Illuminate
- IBM: watsonx Orchestrate
- Zendesk、HubSpot、Oracle も参戦

## キーデータ

- AIエージェント市場: 2025年$78.4億 → 2030年$526.2億（CAGR 46.3%）
- 2026年末までにエンタープライズアプリの40%がAIエージェント搭載（2025年は5%未満）
- マルチエージェントシステムへの問い合わせ: 2024年Q1→2025年Q2で1,445%急増
- SaaSツール乱立による損失: 中小企業で従業員1人あたり年間約$18,000
- 開発者の93%がAI使用、しかし生産性向上は10%で頭打ち
- AIユーザーのツール切替頻度上昇（74%が気づいていない）
- AIエージェントに30+ツール定義 → 判断精度低下
- マルチエージェント: 問題解決45%高速化、精度60%向上（単体比）
- 独立エージェントはエラー17.2倍増幅、中央集権的調整で4.4倍に抑制

## 業界の「常識」（ぶった切り候補）

- 常識1:「SaaSにAIエージェントが載った = SaaSが進化した」
  → 実はこれが間違い。新聞がウェブサイトを作ったのと同じ。既存パラダイムの延長にAIを載せただけ
- 常識2:「たくさんのAIエージェントがあれば生産性が上がる」
  → 実は逆。エージェントスプロール（乱立）で年間$18,000/人の損失。ツール切替疲労は悪化
- 常識3:「UIが優れたSaaSが勝つ」
  → AIエージェントが仕事するなら、UIは無意味。オーケストレーション層を制する者が勝つ

## アドネス哲学との接続ポイント

- Well-being: ツール乱立 = ストレス = Well-beingの敵。統一的な「アドる」体験こそWell-being
- LTR: SaaSのロックイン戦略 vs 長期的に価値を提供し続けるプラットフォーム
- Giver/Taker: SaaS各社はロックインで搾取（Taker）、本当のGiverはオープンな統合を提供する
- 価値提供: 「ツールを使う」→「ゴールを設定する」への価値転換
- アドる: 「ググるとわかる。アドればできる。」= AIエージェント時代の最終形態。ゴール設定→実行の変換装置

## みかみの体験談で使えそうなもの

- 300人組織で複数SaaS（Notion, Slack等）を使っている実感

- VisionToDoやAddnessプロダクトがまさにこの「オーケストレーション層」
