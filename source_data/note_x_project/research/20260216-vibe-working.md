# バイブワーキング（Vibe Working）に関するリサーチメモ

## 概要

「バイブコーディング（Vibe Coding）」は2025年のバズワード。Andrej Karpathyが2025年2月2日にXで提唱し、Collins Dictionaryの2025年Word of the Yearに選ばれた。「やりたいことを自然言語で伝え、AIがコードを生成する」開発スタイル。

2026年、この概念がエンジニアの世界を超えて全ナレッジワーカーに拡大。Anthropic、Microsoftが「バイブワーキング（Vibe Working）」として公式に推進。これが2026年最大の働き方トレンド。

## キーデータ

- 2025年2月2日: Andrej Karpathy（OpenAI共同創業者）がXで「vibe coding」を提唱。450万ビュー超
- 2025年: Collins Dictionary の Word of the Year に「vibe coding」選出
- 2025年: Y Combinator Winter 2025バッチの25%がコードベースの95%をAI生成
- 2026年2月5日: Anthropic Scott White（Head of Product）がCNBCで「vibe working」に言及。Claude Opus 4.6のローンチと共に
- 2025年9月29日: Microsoft Sumit Chauhan（CVP, Office Product Group）がMicrosoft 365 Copilotに「vibe working」を組み込むと発表
- 2026年現在: 開発者の72%がAIツールを毎日使用。コミット済みコード全体の約42%がAI生成
- Goldman Sachsが社内で数百のDevinインスタンスを運用。反復タスクで10-12倍の効率向上
- 国内上場企業の8割以上が「業務の一部でAIを活用」

## バイブコーディング vs バイブワーキングの対比

### バイブコーディング（2025）
- 対象: エンジニア・開発者
- やること: 自然言語でやりたいことを伝え、AIがコードを生成
- ツール: Cursor, Claude Code, GitHub Copilot
- 人間の役割: 方向性の指示、レビュー、品質管理
- Karpathyの名言: "I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works."

### バイブワーキング（2026）
- 対象: 全ナレッジワーカー（営業、人事、企画、経営者、ライター...）
- やること: 自然言語で成果物のイメージを伝え、AIが実行
- ツール: Claude Opus 4.6, Microsoft 365 Copilot Agent Mode, Cowork
- 人間の役割: 「オーケストレーター」— 目標設定、制約定義、品質ゲート
- Scott White: "Everybody has seen this transformation happen with software engineering... I think that we are now transitioning almost into vibe working."

### 共通の本質
- 人間は「オペレーター」から「オーケストレーター」へ
- 生産性の単位が「アプリに費やした時間」→「完成した成果物」に変化
- 自然言語のインテント（意図）→ AIが実行計画を立てて実行
- 人間に残る仕事 = 「ゴールの設定」と「品質の判断」

## 業界の「常識」（ぶった切り候補）

- 常識1: 「AI活用 = プロンプトエンジニアリングを学ぶこと」→ 違う。「何を作りたいか」を明確にすることの方が100倍大事
- 常識2: 「バイブコーディングはエンジニアの話でしょ？」→ もう違う。全職種に波及している
- 常識3: 「AIに仕事を奪われる」→ 違う。「AIに仕事を任せて、人間はゴール設定に集中する」が正解
- 常識4: 「AIツールを使いこなすスキルが必要」→ 最も必要なのは「何を達成したいか」を言語化するスキル

## アドネス哲学との接続ポイント（★超重要）

- **「ググるとわかる。アドればできる。」**: バイブワーキングの本質そのもの。「知る」→「できる」の谷をAIが埋める時代
- **アドる**: バイブワーキングの日本語訳が「アドる」と言っても過言ではない。ゴールを持って→実行に移す→AIと共に最高効率で実現
- **Well-being**: バイブワーキングで人間は「作業」から解放され、「考える」「決める」「関係を紡ぐ」に集中できる = Well-being的な働き方
- **Giver**: 作業コストが激減→価値提供の総量が爆発的に増える→全員がGiverになれる時代
- **価値提供**: AIが実行を担う→人間は「何が価値か」を考えることに集中→価値提供の質が上がる
- **LTR**: AIが雑務を処理→人間は関係性構築に時間を使える→LTRが深まる

## みかみの体験談で使えそうなもの

- 東大中退して起業: 「正解のない問いに向き合う」= ゴール設定力
- 留置所体験: 作業を奪われた環境で「何が大事か」を考えた = バイブワーキングの本質（作業がなくなった時に残るもの）
- 100万円とダンボール机でアドネス創業: 「ゴール」さえあれば始められる
- 非エンジニアで300人組織を率いる: コードは書けないが「何を作るか」は語れる = バイブワーキング経営者
- ClaudeCoworkでnote記事を書いた体験: まさにバイブワーキングの実践例

## 記事の切り口

1. 「2025年はバイブコーディング元年だった。2026年はバイブワーキング元年になる。」— 対比構造で全職種に拡大する警鐘/ビジョン
2. 「バイブコーディング→バイブワーキング→アドる」の三段階で概念を接続
3. 最も大事なスキルは「プロンプトの書き方」じゃない。「ゴールの設定」だ。= アドネスの存在意義
